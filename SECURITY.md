| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security of Leather Earn seriously. If you discover a security vulnerability, please follow these steps:

### How to Report

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via one of the following methods:

1. **Email**: Send details to [security@leather.io](mailto:security@leather.io)
2. **GitHub Security Advisories**: Use the [Security tab](../../security/advisories/new) to privately report a vulnerability

### What to Include

When reporting a vulnerability, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (if available)
- Your contact information for follow-up

### What to Expect

- **Acknowledgment**: We will acknowledge receipt of your vulnerability report within 48 hours
- **Updates**: We will provide regular updates on our progress (at least every 7 days)
- **Timeline**: We aim to:
  - Confirm the vulnerability within 7 days
  - Release a fix within 30 days for critical issues
  - Release a fix within 90 days for moderate issues

### Disclosure Policy

- Please give us reasonable time to address the vulnerability before any public disclosure
- We will credit you in the release notes (unless you prefer to remain anonymous)
- We follow coordinated disclosure practices

## Security Best Practices

When using or contributing to Leather Earn:

1. **Dependencies**: Keep all dependencies up to date
2. **Environment Variables**: Never commit sensitive data or API keys
3. **Code Review**: All changes require review before merging
4. **Testing**: Ensure security-related changes include appropriate tests

## Bug Bounty Program

We do not currently have a bug bounty program, but we deeply appreciate security researchers who responsibly disclose vulnerabilities.

## Questions

If you have questions about this security policy, please contact [security@leather.io](mailto:security@leather.io).
